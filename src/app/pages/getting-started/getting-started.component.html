Getting Started

The best way to quickly integrate Foundation for Sites Components with Angular 6 or 7.

View the Changelog and Release Notes

ngx-foundation works with all core Foundation for Sites components powered by Angular. Original JS components like jQuery and foundation.js are not needed, but we are using markup and the base SCSS provided by Foundation for Sites.

Looking to integrate Bootstrap with your Angular SPA instead? Visit ngx-bootstrap by Valor Software.

Requirements

  • Foundation for Sites v6.4 or higher using SCSS or CSS
  • Angular 6 or 7
  • Angular project using SCSS or CSS
  • "what-input.js" (install optional) useful for accessibility styling

Accessibility

Overview and Limitations

The overall accessibility of any project built with ngx-foundation depends in large part on the author's markup, additional styling, and scripting they've included. However, provided that these have been implemented correctly, it should be perfectly possible to create websites and applications that fulfill WCAG 2.0 (A/AA/AAA), Section 508 and similar accessibility standards and requirements.

Interactive components

Interactive components - such as modal dialogs, dropdown menus and custom tooltips - are designed to work for touch, mouse and keyboard users. Through the use of relevant WAI-ARIA roles and attributes, these components should also be understandable and operable using assistive technologies (such as screen readers).

Authors need to include ARIA roles and attributes to more accurately convey the precise nature of their component.

Color contrast

Some colors that currently make up Foundation for Site's default palette - used throughout the framework for things such as button variations, callout variations, form validation indicators - lead to insufficient color contrast (below the recommended WCAG 2.0 color contrast ratio of 4.5:1) when used against a light background. Authors will need to manually modify/extend these default colors to ensure adequate color contrast ratios.
